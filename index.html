Emmet Extension

Emmet is a very useful extension for text editors that uses abbreviated syntax to convert small snippets of code into full-legnth HTML code and can improve your coding speed significantly.

Syntax Description
“!” + tab key Create HTML boilerplate code
“link” + tab key <link rel="stylesheet" href="">
“a” + tab key <a href="">

Command Line Interface

The Command Line Interface, also called terminal and CLI for short, is a text-based interface used to execute operating system functions. The commands used in this module are for bash in Gitpod, which run on the Linux operating system.

Command Description
“python3 -m http.server” Run web server to preview code (then click open browser)
Command/Ctrl + c Stop web server when running
“clear” Empty display in terminal
“echo <statement>” Print statement in terminal
“echo <statement> <filename>” Prints statement in file
“history” List full history of commands in numbered order
“!1” Run first command in history list
“!2” Run second command in history list
“pwd” Print working directory
“ls” List all visible files in current directory
“ls -la” List all files in current directory including hidden files
“ls <path>” List all visible files in directory
“cd” Change directory
“cd ..” Change directory one level up
“cd <path>” Change to directory
“touch <filename>” Create file with stated file name
“rm <filename>” Remove file
“rm -rf <dirname>” Delete directory and all of its content (no undo!)
“mkdir <dirname>” Create directory with stated directory name
“cp <filename> <destination>” Copy file to stated destination. Using file name as destination duplicates file in current directory a gives it name stated as destination
“mv <filename> <destination>” Move file to stated destination. Using file name as destination renames the file
“cat <filename>” Print out contents of file to the terminal
“diff <filename> <filename>” Print differences between two files

Result explanation:
- “a”: Add
- “c”: Change
- “d”: Delete
- No output: Both files are the same

Example results:
- 1d0: Line 1 in first file needs to be deleted to match line 0 in second empty file
-1c1: Line 1 in first file needs to be changed to match line 1 in second file
“readlink -f <filename>” Print file path of stated file
“nano <filename>” Open up file (or create a new one with stated name) in Nano text editor in the terminal

Step 2:
- command/Ctrl + x: Save file, then “y” to confirm, then enter to confirm file name
“git status” Print status of repository
“git add” Add to staging area
“git add <file name>” Add stated file to staging area
“git add <directory>” Add stated directory to staging area
“git add .” Add all files in working directory to staging area
“git commit -m <commitmessage>” Commit files & folders in staging area with stated commit message
“git rm –cached <filename>” Remove stated file from staging area
“git diff <filename>” Print difference between modified version of file and previous one as it exists in the git repository currently
“git log” List commits that have been made in the repository with associated SHAs
“git reset –hard <first 6 numbers in commit SHA”> Reset repository to stated commit
“git push” Push files & folders in local repository to remote repository
